# モック生成ガイド — Codex imagegen による採用LP モックアップ作成

最終更新: 2026-05-17

---

## 概要

このガイドは、Codex imagegen を使って介護採用LPのモックアップ画像を生成するための手順・ルール・プロンプト設計を定めたものです。
モックは「制作の参考資料」ではなく、**クライアントが承認したら即座にHTML実装のターゲットとなる設計図**です。

---

## モック生成の目的と位置づけ

| 目的 | 詳細 |
|------|------|
| クライアントとの認識合わせ | HTMLを見せる前にビジュアルで合意を取る |
| 実装ターゲットの確定 | 承認後のモックは「これを作る」という仕様書 |
| 手戻りの防止 | 実装後の「イメージと違う」を防ぐ |
| 制作速度の向上 | HTML実装前にデザイン判断を完結させる |

**重要：モックはあくまで実装ターゲットであり、著名サイトや他社サイトのスクリーンショットを参考として使うことは禁止。**

---

## 基本原則

1. **スマホファースト**
   - 最初にスマホ版（375px想定）のモックを生成する
   - PC版はスマホ版が確定してから生成する

2. **セクション構成の固定**
   - 下記「標準セクション構成」に必ず従う

3. **シンプルで実装しやすいデザイン**
   - グラデーションや複雑なアニメーションは避ける
   - CSSで再現しやすいフラットデザインを基本とする

4. **CTAは複数箇所に必ず配置**
   - ファーストビュー / 職種セクション末尾 / フッター直前 の最低3箇所

5. **テキストは実際のコピーを使う**
   - 「ここにテキスト」「Lorem ipsum」等のダミーテキストを使わない
   - ヒアリングシートとコピーライティング結果から実際の文言を入力する

---

## 標準セクション構成

採用LPは以下のセクション順で構成する。変更する場合は担当者承認を必要とする。

```
[01] ヘッダー
     - 施設ロゴ / 施設名
     - 電話番号（tel:リンク）
     - 「今すぐ応募する」ボタン（右上）

[02] ファーストビュー（ヒーローセクション）
     - 大きなキャッチコピー（施設の魅力・採用テーマ）
     - サブキャッチコピー
     - CTAボタン（「今すぐ応募する」「求人を見る」）
     - 施設の写真またはイラスト背景

[03] 求人情報サマリー（ひと目でわかる3〜4項目）
     - 月給 / 時給
     - 雇用形態
     - 勤務地
     - 未経験歓迎 / 夜勤なし等の強み

[04] 職種一覧
     - 採用したい職種をカード形式で表示
     - 各カードに「詳細を見る」リンク

[05] 施設紹介
     - 施設の特色・こだわり・雰囲気
     - 写真（1〜3枚）

[06] 先輩スタッフの声（あれば）
     - 顔写真（任意）/ 名前（ニックネーム可）/ コメント

[07] 仕事の1日の流れ
     - タイムライン形式

[08] 職場環境・待遇
     - 福利厚生・研修・資格支援等をアイコン付きで

[09] よくある質問（FAQ）
     - アコーディオン形式（HTML上）を意識したデザイン
     - 最低5件

[10] 採用の流れ
     - 「応募 → 書類確認 → 面接 → 内定」等のステップ図

[11] 応募CTA（最終）
     - 「今すぐ応募する」ボタン（大きく・目立つ色）
     - 電話番号
     - LINE（あれば）

[12] フッター
     - 施設名・住所・電話番号・コピーライト
```

## カラーパレット設計ルール

| 用途 | 推奨カラー | 理由 |
|------|-----------|------|
| CTAボタン | オレンジ系（#FF8C00 等）または 深い緑（#2E7D32 等） | 介護業界の親しみやすさ・ケアのイメージ |
| 見出し・テキスト | 濃いグレー（#222222 等） | 読みやすさ |
| 背景 | 白（#FFFFFF）または 淡いグレー（#F8F8F8） | 清潔感 |
| アクセント | メインカラーと補色 | セクション区切りや強調に使用 |
| 避けるべき色 | 病院的な寒色のみ（#00B4D8のみ等） | 冷たい印象を避ける |

※ クライアントが既存のブランドカラーを持つ場合はそちらを優先する

---

## スペーシング・余白のルール

| 場所 | 推奨余白 |
|------|---------|
| セクション間の上下余白 | 60〜80px |
| コンテナの左右余白（スマホ） | 16〜20px |
| 見出しと本文の間隔 | 16〜24px |
| カード内のパディング | 20〜24px |
| CTAボタンのパディング | 縦14〜18px / 横30〜40px |

---

## CTAボタンのデザインルール

| 要素 | 設計 |
|------|------|
| テキスト | 「今すぐ応募する」「まずは見学だけでもOK」等の行動を促す文言 |
| 色 | ページ内で最も目立つ色（CTAのみに使うアクセントカラー） |
| 形状 | 角丸（border-radius: 6〜8px）または 完全な丸み（pill型） |
| サイズ（スマホ） | 幅は画面の80〜90%、高さは50〜56px |
| サイズ（PC） | 幅は内容に合わせて、高さは48〜56px |
| 補足テキスト | ボタンの下に「※ 見学だけでも大丈夫です」等の安心文言 |

---

## カードレイアウトのルール（職種一覧・特徴等）

| 要素 | 設計 |
|------|------|
| スマホ時 | 1カラム（縦積み） |
| タブレット時 | 2カラム |
| PC時 | 3カラム（最大4カラム） |
| カード影 | `box-shadow: 0 2px 8px rgba(0,0,0,0.1)` 程度 |
| カード角丸 | `border-radius: 8〜12px` |
| カード内のアイコン | 職種別アイコン（SVG）またはシンプルなピクトグラム |

---

## HTML実装しやすいデザインにするためのルール

以下の要素はHTML/CSSで実装が困難なため、モックには含めない：

| NG要素 | 代替案 |
|--------|--------|
| 複雑な曲線の背景（SVGで細かいウェーブ等） | シンプルな直線・緩やかな斜めカット |
| テキストの曲線配置 | 通常の水平配置 |
| 複雑なアニメーション | ホバーエフェクト程度（CSSで実現可能なもの） |
| 重なり合う要素が5層以上 | 最大2〜3層に抑える |
| 自由変形した図形 | 長方形・円・三角形の組み合わせ |

## Codex imagegen へのプロンプトテンプレート

### スマホ版 ファーストビュー モック

```
介護施設の採用ランディングページ、スマートフォン版（幅375px想定）のファーストビュースクリーンショット。
以下の仕様で生成すること：

- ヘッダー：施設ロゴ（左）、電話番号（中央）、「今すぐ応募」ボタン（右）
- ヒーローエリア：施設名とキャッチコピー「○○」が大きく表示される
- サブキャッチ：「○○」が細字で表示
- CTAボタン：オレンジ色の大型ボタン「今すぐ応募する」
- 背景：施設の明るい室内写真（白・温かみのある雰囲気）
- カラー：#FF7A00（CTA）、#FFFFFF（背景）、#333333（テキスト）
- フォント：丸ゴシック系、読みやすく日本語向き
- 全体トーン：清潔感があり、親しみやすい介護施設のイメージ

UIは実際のHTMLページのスクリーンショット風に見えるようにすること。
```

### スマホ版 全体レイアウト モック

```
介護施設の採用ランディングページ、スマートフォン版（幅375px）の全体レイアウト俯瞰図。
以下のセクションをすべて縦に含めること：

01. ヘッダー（ロゴ・電話・CTAボタン）
02. ファーストビュー（キャッチコピー・写真・CTAボタン）
03. 求人情報サマリー（3項目：月給・勤務形態・勤務地）
04. 職種カード一覧（介護士・ケアマネ・看護師）
05. 施設紹介（テキスト+写真）
06. よくある質問（5件、シンプルなリスト形式）
07. 応募CTA（ボタン・電話番号）
08. フッター（施設名・住所）

実際のHTMLページのスクリーンショット風で生成すること。
カラー：CTAボタン #FF7A00、背景白系、テキスト濃グレー
```

## モック生成後の確認リスト

Codex imagegen でモックを生成したら、以下を確認してからクライアントへ送付する。

| 確認項目 | チェック |
|---------|---------|
| 全セクションが揃っているか | □ |
| CTAボタンが3箇所以上あるか | □ |
| ダミーテキストが残っていないか | □ |
| 電話番号・施設名が正確か | □ |
| スマホ版とPC版の両方があるか | □ |
| HTML実装が困難な要素が含まれていないか | □ |

---

## クライアントへの送付方法

1. 画像をPNGまたはJPEGで書き出す
2. Google Drive のフォルダに格納（クライアントと共有）
3. モックへのコメント方法をクライアントに案内する
   - Google ドキュメントにスクリーンショットを貼り付けてコメント
   - または画像に直接矢印・書き込みをしてもらう形式

---

## モック修正のルール

| 修正種別 | 対応方針 |
|---------|---------|
| テキストの微調整 | コピーを修正して再生成 |
| セクション順序の変更 | 再生成が必要。プロンプトを更新 |
| カラーの変更 | プロンプトのカラー指定を更新して再生成 |
| セクション全体の追加・削除 | 担当者確認のうえ再生成 |
| デザイン全体の方向性変更 | 追加費用の対象となる場合あり（要確認） |

## ファイル命名規則

```
mock_[施設名略称]_sp_v[バージョン].png      # スマホ版
mock_[施設名略称]_pc_v[バージョン].png      # PC版
mock_[施設名略称]_sp_v[バージョン]_r[修正回].png  # 修正版

例：
mock_sakura_sp_v1.png
mock_sakura_pc_v1.png
mock_sakura_sp_v1_r1.png
```
