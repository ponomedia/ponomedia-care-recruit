# Codex コードレビューチェックリスト — 介護採用まるごと導線パック

最終更新: 2026-05-17

---

## 概要

このチェックリストは、HTML/CSS実装完了後にCodexを用いてコードレビューを実施する際の確認項目です。
Codexに対するレビュー依頼プロンプトと、確認項目・判定基準を定義しています。

---

## Codexへのレビュー依頼プロンプトテンプレート

以下のプロンプトをCodexに入力し、レビューを実施します。

```
以下のHTMLファイルをレビューしてください。
対象は介護施設の採用ランディングページです。

チェックすべき観点：
1. コード品質（無駄なコード・構造的な問題）
2. UIの問題（レイアウト崩れ・視認性の問題）
3. モバイル最適化（レスポンシブ対応）
4. 基本的なSEO（タイトル・メタ・見出し構造）
5. アクセシビリティ（alt属性・ラベル・コントラスト）
6. 構造化データ（JobPosting JSON-LDの整合性）
7. 応募導線（CTAボタンの設置・フォームリンク）
8. CTAの配置（ファーストビュー・中段・末尾への設置）
9. セキュリティ（外部リンクのrel・フォームのaction等）
10. 未使用コード・重複スタイル
11. テキスト内容の矛盾・誤字（確認できる範囲で）
12. モックアップ画像との再現精度（提供した画像と比較）

問題点を「カテゴリ」「問題箇所（行番号等）」「重大度（高/中/低）」「修正方法」の形式で出力してください。
最後に改善提案サマリーを記述してください。
```

## チェック項目・判定基準

---

### 1. コード品質

| チェック項目 | 重大度 | 判定基準 |
|------------|--------|---------|
| HTMLの構造が意味論的に正しい（div乱用ではなくheader/main/section/footer等の使用） | 中 | 適切なセマンティックタグを使用している |
| CSSに重複するスタイル定義がない | 低 | 同じプロパティが同じセレクターで複数回定義されていない |
| インラインスタイルが多用されていない | 低 | インラインstyleはなるべく外部CSS・style要素で管理されている |
| コメントが最低限あり、コードが読みやすい | 低 | 主要なセクション分けにコメントがある |
| 不要なJavaScript（使っていないスクリプト）が読み込まれていない | 中 | head/bodyに不要なscriptタグがない |

**判定：** OK / NG / 要確認
**コメント：**

---

### 2. UIの問題

| チェック項目 | 重大度 | 判定基準 |
|------------|--------|---------|
| レイアウトが崩れている箇所がない | 高 | 全セクションで要素が想定位置に表示されている |
| フォントサイズが本文16px以上・見出しは適切な階層 | 中 | h1 > h2 > h3 の順でサイズが調整されている |
| 色のコントラストが十分（テキストと背景） | 中 | WCAG AAレベル（コントラスト比4.5:1以上）を目安に確認 |
| 画像のalt属性が適切に設定されている | 中 | 装飾画像はalt=""、情報画像は意味のある説明文 |
| ボタンのサイズがタップしやすい（最小44×44px） | 高 | スマホでのタップ操作を考慮したサイズ |

**判定：** OK / NG / 要確認
**コメント：**

---

### 3. モバイル最適化

| チェック項目 | 重大度 | 判定基準 |
|------------|--------|---------|
| viewport metaタグが正しく設定されている | 高 | `<meta name="viewport" content="width=device-width, initial-scale=1">` が存在する |
| メディアクエリが設定されており、ブレークポイントが適切 | 高 | 768px / 1024px等でレイアウトが切り替わる |
| スマホで横スクロールが発生しない | 高 | `overflow-x: hidden` や `max-width: 100%` が適切に設定されている |
| スマホでテキストが折り返し、読みやすい幅に収まっている | 中 | 行長が適切（スマホで1行40〜50文字目安） |
| タッチターゲットの間隔が狭すぎない | 中 | 隣接するリンク・ボタンの間隔が8px以上 |

**判定：** OK / NG / 要確認
**コメント：**

---

### 4. 基本的なSEO

| チェック項目 | 重大度 | 判定基準 |
|------------|--------|---------|
| title タグが設定されており、施設名・職種・地域名が含まれている | 高 | 例：「介護職員 募集中 ｜ ○○施設（○○市）」 |
| meta description が設定されており、80〜120文字 | 中 | 求人の概要・魅力・施設名が含まれている |
| h1 タグが1つだけ存在する | 高 | ページ内でh1は1つのみ |
| 見出し階層（h1〜h3）が論理的な順序になっている | 中 | h2の下にh3、h1の下にh2という順序が守られている |
| 画像に alt属性が設定されている | 中 | 空alt=""でも設定されており、意味のある画像は説明文あり |
| 正規URL（canonical）の設定 | 低 | 必要に応じてcanonical linkが設定されている |

**判定：** OK / NG / 要確認
**コメント：**

---

### 5. アクセシビリティ

| チェック項目 | 重大度 | 判定基準 |
|------------|--------|---------|
| フォームの各入力欄に label が紐付いている | 中 | `<label for="name">` と `<input id="name">` が対応している |
| ボタンに aria-label または テキストが設定されている | 中 | アイコンのみのボタンには aria-label が必要 |
| 色のみで情報が伝わらない設計（色盲対応） | 低 | エラー表示はアイコン・テキストも併用 |
| キーボードでのフォーカス移動が自然な順序 | 低 | tabindex が適切、または自然なDOM順でフォーカスが移動 |
| lang 属性が html タグに設定されている | 低 | `<html lang="ja">` が設定されている |

**判定：** OK / NG / 要確認
**コメント：**

---

### 6. 構造化データ（JobPosting JSON-LD）

| チェック項目 | 重大度 | 判定基準 |
|------------|--------|---------|
| JSON-LD の script タグが head または body に存在する | 高 | `<script type="application/ld+json">` が確認できる |
| `@type: "JobPosting"` が正しく設定されている | 高 | Googleの仕様通りのtype名 |
| 必須プロパティが揃っている（title / description / datePosted / validThrough / hiringOrganization / jobLocation） | 高 | [Google仕様](https://developers.google.com/search/docs/appearance/structured-data/job-posting) に照合 |
| datePosted / validThrough が ISO 8601 形式 | 中 | 例：`"2026-05-17"` |
| baseSalary が正しい形式で設定されている | 中 | `MonetaryAmount` 型で value / currency / unitText が設定されている |
| Googleリッチリザルトテストにエラーがない | 高 | テストURLでエラー0件を確認 |

**判定：** OK / NG / 要確認
**コメント：**

### 7. 応募導線

| チェック項目 | 重大度 | 判定基準 |
|------------|--------|---------|
| CTAボタンが1ページ内に複数設置されている（最低3箇所） | 高 | ファーストビュー / 中段（条件セクション付近）/ フッター付近 |
| CTAボタンのテキストが行動を促す表現（「今すぐ応募する」等） | 中 | 「こちら」「詳細はこちら」などの曖昧表現ではない |
| CTAボタンがフォームまたは電話番号に正しくリンクされている | 高 | クリック先がフォームページ or tel:リンク |
| 電話番号が `tel:` リンクになっている | 高 | `<a href="tel:0000000000">` の形式 |
| フォームの action / method が正しく設定されている | 高 | Google フォームのURLまたは適切なエンドポイント |

**判定：** OK / NG / 要確認
**コメント：**

### 8. CTAの配置

| チェック項目 | 重大度 | 判定基準 |
|------------|--------|---------|
| ファーストビュー（スクロールなしで見える範囲）にCTAボタンがある | 高 | スマホ・PC両方で確認 |
| 長いページの中段（条件・FAQ付近）にCTAボタンがある | 中 | 読み進めた求職者が応募しやすい位置 |
| ページ末尾（フッター直前）にCTAボタンがある | 中 | 最後まで読んだ求職者へのフォロー |
| スマホのフッターに固定CTAバー（sticky CTA）がある（あれば） | 低 | 実装している場合は他のコンテンツと被っていないか確認 |

**判定：** OK / NG / 要確認
**コメント：**

### 9. セキュリティ

| チェック項目 | 重大度 | 判定基準 |
|------------|--------|---------|
| 外部リンクに `rel="noopener noreferrer"` が設定されている | 中 | `target="_blank"` を使う外部リンクには必須 |
| フォームの送信先が正しい（悪意のあるエンドポイントでない） | 高 | Google フォームのURLまたは承認済みのエンドポイント |
| メールアドレスがテキストのまま露出していない（スパム対策） | 低 | obfuscation や フォーム経由での受け取り推奨 |
| 外部CDN（Font等）のSRI（Subresource Integrity）が設定されている（可能な場合） | 低 | 外部リソース使用時はintegrity属性を検討 |

**判定：** OK / NG / 要確認
**コメント：**

### 10. 未使用コード・重複スタイル

| チェック項目 | 重大度 | 判定基準 |
|------------|--------|---------|
| 使用されていないCSSクラスが大量に含まれていない | 低 | 未使用クラスが5件以上ある場合は整理推奨 |
| コメントアウトされたコードが残っていない | 低 | 本番コードにデバッグ用コメントが残っていない |
| 同じスタイルが複数セレクターに重複定義されていない | 低 | CSSが整理されているか確認 |

**判定：** OK / NG / 要確認
**コメント：**

### 11. テキスト内容の矛盾

| チェック項目 | 重大度 | 判定基準 |
|------------|--------|---------|
| ヘッダーとフッターで施設名・電話番号の表記が一致している | 高 | 施設名の表記ゆれがない |
| 給与情報が複数箇所ある場合、数値が一致している | 高 | 「月給〇〇万円〜」の数値が全セクションで同じ |
| 職種名の表記が全ページで統一されている | 中 | 「介護士」「介護スタッフ」等の表記ゆれがない |
| 誇大・違法表現が含まれていない | 高 | `docs/service-definition.md` の禁止表現と照合 |

**判定：** OK / NG / 要確認
**コメント：**

### 12. モックアップ再現精度

| チェック項目 | 重大度 | 判定基準 |
|------------|--------|---------|
| 全セクションの順序がモックと一致している | 高 | セクションの並び順が承認済みモックと同じ |
| 主要なカラー（背景色・テキスト色・ボタン色）がモックと一致 | 中 | カラーコードがモックのデザインと合っている |
| フォントの太さ・サイズの大小関係がモックと一致 | 中 | 見出し・本文・キャプションの視覚的な階層が再現されている |
| カード・グリッドレイアウトがモックと一致 | 中 | 職種カード・特徴カードの並び方が再現されている |
| CTAボタンの色・テキスト・形状がモックと一致 | 高 | ボタンデザインが承認済みのモックと同じ |

**判定：** OK / NG / 要確認
